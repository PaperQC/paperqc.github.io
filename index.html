PaperQC

PaperQC will be a downloadable quality-control toolkit for evaluating scientific papers. It will run locally as a Python package and as a Nextflow pipeline, helping researchers screen literature before it becomes the basis for experiments, reviews or investment decisions.

What it will check

Quality signals for messy scientific literature.

PaperQC will start with transparent metadata and citation checks, then expand toward deeper content analysis as the project matures.

Bibliometric context

It will summarize publication metadata, journal context, citation patterns and unusual bibliographic signals.

Citation network risk

It will flag retracted citations, clusters of suspiciously narrow citation behavior and excessive self-citation patterns.

Author and venue history

It will surface retraction history, journal warnings and other context.

Future content checks

Later versions will explore p-hacking signals, image integrity, tortured phrases, AI-generated text and method inconsistencies.

How it will work

From paper identifier to reproducible report.

Input

Bring papers from existing tools

Users will pass PDFs, RIS, BibTeX, DOI lists or metadata tables from literature searches, Zotero libraries and review workflows.

Analyze

Run locally

PaperQC will be a Python package and Nextflow pipeline.

Report

Review ranked evidence, not ambiguous scores

Outputs will include readable reports and machine-friendly tables that explain why each paper, citation or venue was flagged.

Science moves faster when researchers find weak links
before building on them.

Retractions, irreproducible studies and manipulated results can redirect whole fields long before corrections catch up. PaperQC will help teams inspect that risk early, at the level of the paper collection they are about to trust.

The project will favor reproducibility over polish: local runs, open code, inspectable data outputs and documented assumptions. The goal will be to support human judgment, not replace it.

Open-source roadmap

A practical path to the first release.

PaperQC does not exist yet as a public package. The first release will focus on dependable, explainable checks that can be extended by the community.

Phase 1

Command-line prototype

A Python package will parse paper collections, retrieve metadata and generate the first citation and bibliometric quality reports.

Phase 2

Pipeline-scale analysis

A Nextflow workflow will make large literature screens reproducible across laptops, servers and shared compute.

Phase 3

Community modules

The project will add modular checks for PubPeer signals, journal warnings, p-value patterns, figures and paper text.

People

Built by scientists for scientists.

The project will be shaped by researchers who understand how literature quality affects experiments, reviews and funding decisions.

PaperQC will be released as an open-source terminal toolkit.
